1. Que Hace el Sistema

El sistema permite tomar pedidos online y registrar altas de nuevos clientes de forma ordenada en Google Sheets.

  • El cliente busca por direccion o telefono.
  • El sistema encuentra el cliente y muestra solo sus horarios habilitados.
  • El cliente arma el pedido y se guarda automaticamente en la hoja Pedidos.
  • Las altas de nuevos clientes se guardan en AltasAutomaticas.

2. Pantallas y Flujo de Pedidos

2.1 Indicador de paso (pastilla superior)

  • INICIO: pantalla de busqueda.
  • 1 DE 2: seleccion de horario.
  • 2 DE 2: armado del pedido.
  • FINALIZADO: confirmacion de pedido enviado.

2.2 Pantalla INICIO

  • Campo: Direccion completa o telefono.
  • Boton principal: Buscar mi direccion.
  • Boton secundario: Hablar con un asesor (WhatsApp).

La busqueda tolera variaciones comunes, por ejemplo:

  • Sin espacio entre calle y numero: BARADERO6140.
  • Error tipico: BRADERO6140 (se interpreta como BARADERO).

2.3 Pantalla 1 DE 2 (horarios)

  • Muestra los horarios habilitados para ese cliente.
  • Si ninguno aplica, usar: Ninguno, hablar con un asesor.
  • Se puede cambiar usuario con el boton No soy yo / Cambiar direccion.

2.4 Pantalla 2 DE 2 (productos)

  • Cards con imagen, precio y cantidad.
  • El nombre del producto esta oculto en la card para priorizar imagen.
  • Se puede agregar comentario de entrega.

2.5 Pantalla FINALIZADO

  • Muestra resumen de pedido.
  • Permite iniciar un nuevo pedido.

3. Google Sheets: Estructura y Reglas

Importante: No cambiar nombres de pestañas ni encabezados de fila 1. Si se cambian, el sistema deja de mapear datos correctamente.
Hoja Uso Quien la edita
Clientes Base de clientes, direccion, telefono, lista de precio y horarios asignados Operacion
Horarios Catalogo de horarios disponibles Operacion
ProductosPrecios Productos, imagenes y precios por lista Operacion
Pedidos Pedidos generados desde la web Automatico + control operativo
AltasAutomaticas Solicitudes de alta de nuevos clientes Automatico + seguimiento comercial

3.1 Reglas de carga recomendadas

  • Direcciones en MAYUSCULAS.
  • Telefonos sin simbolos extra (solo numeros).
  • Campos Activo usar valores tipo SI/NO o TRUE/FALSE segun lista.
  • Para horarios por cliente usar columnas Horario 1 a Horario 4.

4. Flujo de Altas Automaticas

4.1 Que datos recibe

  • Direccion
  • Localidad
  • Codigo de area
  • Celular

4.2 Como se guarda

  • Se inserta una fila nueva en AltasAutomaticas.
  • Direccion se guarda en mayusculas automaticamente.
  • Se aplica formato visual (header azul, filtro, intercalado de filas).
  • Localidad y Estado quedan con validacion de lista.

4.3 Estados sugeridos para seguimiento

  • NUEVO
  • PENDIENTE CONTACTO
  • PRIMER INTENTO
  • SEGUNDO INTENTO
  • CONTACTADO
  • INTERESADO
  • NO INTERESADO
  • NO RESPONDE
  • DATOS INCOMPLETOS
  • ALTA CONCRETADA

5. Tareas Operativas (Paso a Paso)

5.1 Agregar un nuevo cliente habilitado para pedidos

  1. Abrir hoja Clientes.
  2. Crear fila nueva con: id, direccion, localidad, telefono, activo, lista de precio.
  3. Completar Horario 1..4 con IDs validos de la hoja Horarios.
  4. Verificar que Activo este en SI.

5.2 Agregar o editar productos

  1. Abrir hoja ProductosPrecios.
  2. Editar precio lista 1 y/o lista 2.
  3. Verificar URL de imagen.
  4. Confirmar estado activo por lista.

5.3 Eliminar un pedido cargado por error

  1. Ir a hoja Pedidos.
  2. Seleccionar fila completa.
  3. Clic derecho > Eliminar fila.

6. Que Hacer Si Algo Falla

6.1 No encuentra direccion que deberia existir

  • Revisar que el cliente este ACTIVO en Clientes.
  • Verificar que la direccion este bien escrita.
  • Probar buscar por telefono.

6.2 No llegan pedidos a la hoja

  • Validar URL de API en src/config.js.
  • Confirmar que el Apps Script este desplegado como Web App.
  • Revisar permisos del despliegue (acceso publico segun configuracion actual).

6.3 Cambie codigo y no lo veo en la web

  • GitHub Pages puede tardar 1 a 3 minutos.
  • Probar en modo incognito.
  • Usar URL con cache-buster: ?ts=fecha-hora.

7. Publicacion y Actualizaciones

7.1 Frontend (GitHub Pages)

  1. Guardar cambios.
  2. Hacer commit y push a rama main.
  3. Esperar deploy automatico de GitHub Pages.

7.2 Backend (Google Apps Script)

  1. Pegar cambios en apps-script/Code.gs.
  2. Deploy > Manage Deployments > New Version.
  3. Confirmar.

Si no se redeploya Apps Script, los cambios de backend no impactan aunque el sitio web este actualizado.

8. Checklist Rapido Diario

  • Verificar si entraron pedidos nuevos en Pedidos.
  • Revisar nuevas filas en AltasAutomaticas.
  • Actualizar estado de altas segun gestion telefonica.
  • Controlar que precios y horarios vigentes esten correctos.
